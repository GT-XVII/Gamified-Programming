Timer reference indicating when to send the next ping or when the most recent ping will be considered lost.

Indicates that the keepalive timer ran out while there were no active calls, and a ping should be sent the next time a call starts.

Parameters
remoteNameName of the remote server, if it is not the same as the subchannel address, i.e. if connecting through an HTTP CONNECT proxy.

◆ maybeStartKeepalivePingTimer()

Http2Transport::maybeStartKeepalivePingTimer ( )
inline

Starts the keepalive ping timer if appropriate. If the timer already ran out while there were no active requests, instead send a ping immediately. If the ping timer is already running or a ping is currently in flight, instead do nothing and wait for them to resolve.
