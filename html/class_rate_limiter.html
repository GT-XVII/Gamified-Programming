Detailed Description

A helper that uses the Token Bucket algorithm to rate limit the number of operations that can be made in a second.

Before a given request containing a number of operations can proceed, RateLimiter determines doing so stays under the provided rate limits. It can also determine how much time is required before a request can be made.

RateLimiter can also implement a gradually increasing rate limit. This is used to enforce the 500/50/5 rule (https://firebase.google.com/docs/firestore/best-practices#ramping_up_traffic).
