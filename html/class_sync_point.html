Detailed Description

SyncPoint represents a single location in a SyncTree with 1 or more event registrations, meaning we need to maintain 1 or more Views at this location to cache server data and raise appropriate events for server changes and user writes (set, transaction, update).

It's responsible for:

  • Maintaining the set of 1 or more views necessary at this location (a SyncPoint with 0 views should be removed).
  • Proxying user / server operations to the views as appropriate (i.e. applyServerOverwrite, applyUserOverwrite, etc.)
