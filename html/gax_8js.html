function constructSettings ( serviceName,
clientConfig,
configOverrides,
retryNames,
otherArgs )

Constructs a dictionary mapping method names to CallSettings.

The clientConfig parameter is parsed from a client configuration JSON file of the form: 

{
  "interfaces": {
    "google.fake.v1.ServiceName": {
      "retry_codes": {
        "idempotent": ["UNAVAILABLE", "DEADLINE_EXCEEDED"],
        "non_idempotent": []
      },
      "retry_params": {
        "default": {
          "initial_retry_delay_millis": 100,
          "retry_delay_multiplier": 1.2,
          "max_retry_delay_millis": 1000,
          "initial_rpc_timeout_millis": 2000,
          "rpc_timeout_multiplier": 1.5,
          "max_rpc_timeout_millis": 30000,
          "total_timeout_millis": 45000
        }
      },
      "methods": {
        "CreateFoo": {
          "retry_codes_name": "idempotent",
          "retry_params_name": "default"
        },
        "Publish": {
          "retry_codes_name": "non_idempotent",
          "retry_params_name": "default",
          "bundling": {
            "element_count_threshold": 40,
            "element_count_limit": 200,
            "request_byte_threshold": 90000,
            "request_byte_limit": 100000,
            "delay_threshold_millis": 100
          }
        }
      }
    }
  }
}
Parameters
{String}serviceName - The fully-qualified name of this service, used as a key into the client config file (in the example above, this value should be 'google.fake.v1.ServiceName').
{Object}clientConfig - A dictionary parsed from the standard API client config file.
{Object}configOverrides - A dictionary in the same structure of client_config to override the settings.
{Object.<string,string[]>}retryNames - A dictionary mapping the strings referring to response status codes to objects representing those codes.
{Object}otherArgs - the non-request arguments to be passed to the API calls.
Returns
{Object} A mapping from method name to CallSettings, or null if the service is not found in the config.

Parameters to the exponential backoff algorithm for retrying.

Parameters
{number}initialRetryDelayMillis - the initial delay time, in milliseconds, between the completion of the first failed request and the initiation of the first retrying request.
{number}retryDelayMultiplier - the multiplier by which to increase the delay time between the completion of failed requests, and the initiation of the subsequent retrying request.
{number}maxRetryDelayMillis - the maximum delay time, in milliseconds, between requests. When this value is reached, retryDelayMultiplier will no longer be used to increase delay time.
{number}initialRpcTimeoutMillis - the initial timeout parameter to the request.
{number}rpcTimeoutMultiplier - the multiplier by which to increase the timeout parameter between failed requests.
{number}maxRpcTimeoutMillis - the maximum timeout parameter, in milliseconds, for a request. When this value is reached, rpcTimeoutMultiplier will no longer be used to increase the timeout.
{number}totalTimeoutMillis - the total time, in milliseconds, starting from when the initial request is sent, after which an error will be returned, regardless of the retrying attempts made meanwhile.
Returns
{BackoffSettings} a new settings.
