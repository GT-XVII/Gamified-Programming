◆ WATCH_IDLE_TIMEOUT_MS

exports WATCH_IDLE_TIMEOUT_MS = 120 * 1000

Idle timeout used to detect Watch streams that stall (see https://github.com/googleapis/nodejs-firestore/issues/1057, b/156308554). Under normal load, the Watch backend will send a TARGET_CHANGE message roughly every 30 seconds. As discussed with the backend team, we reset the Watch stream if we do not receive any message within 120 seconds.
