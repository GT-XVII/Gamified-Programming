The throttled function comes with a {\ttfamily cancel} method to cancel delayed {\ttfamily func} invocations and a {\ttfamily flush} method to immediately invoke them. Provide {\ttfamily options} to indicate whether {\ttfamily func} should be invoked on the leading and/or trailing edge of the {\ttfamily wait} timeout. The {\ttfamily func} is invoked with the last arguments provided to the throttled function. Subsequent calls to the throttled function return the result of the last {\ttfamily func} invocation.

{\bfseries{Note\+:}} If {\ttfamily leading} and {\ttfamily trailing} options are {\ttfamily true}, {\ttfamily func} is invoked on the trailing edge of the timeout only if the throttled function is invoked more than once during the {\ttfamily wait} timeout.

If {\ttfamily wait} is {\ttfamily 0} and {\ttfamily leading} is {\ttfamily false}, {\ttfamily func} invocation is deferred until to the next tick, similar to {\ttfamily set\+Timeout} with a timeout of {\ttfamily 0}.

See \href{https://css-tricks.com/debouncing-throttling-explained-examples/}{\texttt{ David Corbacho\textquotesingle{}s article}} for details over the differences between {\ttfamily \+\_\+.\+throttle} and {\ttfamily \+\_\+.\+debounce}.
