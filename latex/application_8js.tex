Register the given template engine callback {\ttfamily fn} as {\ttfamily ext}.

By default will {\ttfamily \doxylink{__stream__duplex_8js_a472985ef50ed84466f5fce48a490c29b}{require()}} the engine based on the file extension. For example if you try to render a "{}foo.\+ejs"{} file Express will invoke the following internally\+: \begin{DoxyVerb}app.engine('ejs', require('ejs').__express);
\end{DoxyVerb}


For engines that do not provide {\ttfamily .\+\_\+\+\_\+express} out of the box, or if you wish to "{}map"{} a different extension to the template engine you may use this method. For example mapping the EJS template engine to "{}.\+html"{} files\+: \begin{DoxyVerb}app.engine('html', require('ejs').renderFile);
\end{DoxyVerb}


In this case EJS provides a {\ttfamily .render\+File()} method with the same signature that Express expects\+: {\ttfamily (path, options, callback)}, though note that it aliases this method as {\ttfamily ejs.\+\_\+\+\_\+express} internally so if you\textquotesingle{}re using "{}.\+ejs"{} extensions you don\textquotesingle{}t need to do anything.

Some template engines do not follow this convention, the \href{https://github.com/tj/consolidate.js}{\texttt{ Consolidate.\+js}} library was created to map all of node\textquotesingle{}s popular template engines to follow this convention, thus allowing them to work seamlessly within Express.
