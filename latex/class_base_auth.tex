Base\+Auth\+::delete\+Users (\begin{DoxyParamCaption}\item[{}]{uids}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}

Deletes the users specified by the given uids.

Deleting a non-\/existing user won\textquotesingle{}t generate an error (i.\+e. this method is idempotent.) Non-\/existing users are considered to be successfully deleted, and are therefore counted in the {\ttfamily Delete\+Users\+Result.\+success\+Count} value.

Only a maximum of 1000 identifiers may be supplied. If more than 1000 identifiers are supplied, this method throws a \doxylink{class_firebase_auth_error}{Firebase\+Auth\+Error}.

This API is currently rate limited at the server to 1 QPS. If you exceed this, you may get a quota exceeded error. Therefore, if you want to delete more than 1000 users, you may need to add a delay to ensure you don\textquotesingle{}t go over this limit.

Verifies a Firebase session cookie. Returns a Promise with the cookie claims. Rejects the promise if the cookie could not be verified.

If {\ttfamily check\+Revoked} is set to true, first verifies whether the corresponding user is disabled\+: If yes, an {\ttfamily auth/user-\/disabled} error is thrown. If no, verifies if the session corresponding to the session cookie was revoked. If the corresponding user\textquotesingle{}s session was invalidated, an {\ttfamily auth/session-\/cookie-\/revoked} error is thrown. If not specified the check is not performed.

See {\ttfamily \doxylink{}{\texorpdfstring{$\vert$}{|} Verify Session Cookies}} for code samples and detailed documentation
