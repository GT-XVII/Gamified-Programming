Load balancer that attempts to connect to each backend in the address list in order, and picks the first one that connects, using it for every request. 
\begin{DoxyParams}{Parameters}
{\em channel\+Control\+Helper} & {\ttfamily Channel\+Control\+Helper} instance provided by this load balancer\textquotesingle{}s owner. \\
\hline
\end{DoxyParams}
The list of subchannels this load balancer is currently attempting to connect to.

The current connectivity state of the load balancer.

The index within the {\ttfamily subchannels} array of the subchannel with the most recently started connection attempt.

The currently picked subchannel used for making calls. Populated if and only if the load balancer\textquotesingle{}s current state is READY. In that case, the subchannel\textquotesingle{}s current state is also READY.

Listener callback attached to each subchannel in the {\ttfamily subchannels} list while establishing a connection.

The LB policy enters sticky TRANSIENT\+\_\+\+FAILURE mode when all subchannels have failed to connect at least once, and it stays in that mode until a connection attempt is successful. While in sticky TF mode, the LB policy continuously attempts to connect to all of its subchannels.

The most recent error reported by any subchannel as it transitioned to TRANSIENT\+\_\+\+FAILURE.
