\doxysubsection{Detailed Description}
\doxylink{class_sync_tree}{Sync\+Tree} is the central class for managing event callback registration, data caching, views (query processing), and event generation. There are typically two \doxylink{class_sync_tree}{Sync\+Tree} instances for each Repo, one for the normal Firebase data, and one for the .info data.

It has a number of responsibilities, including\+:
\begin{DoxyItemize}
\item Tracking all user event callbacks (registered via add\+Event\+Registration() and remove\+Event\+Registration()).
\item Applying and caching data changes for user \doxylink{database-compat_2dist_2index_8standalone_8js_a0e055d5e5250182755177897e8d99f7c}{set()}, transaction(), and \doxylink{cipher_8js_af2e4dc57151c84e08431510ddacfb9c6}{update()} calls (apply\+User\+Overwrite(), apply\+User\+Merge()).
\item Applying and caching data changes for server data changes (apply\+Server\+Overwrite(), apply\+Server\+Merge()).
\item Generating user-\/facing events for server and user changes (all of the apply\texorpdfstring{$\ast$}{*} methods return the set of events that need to be raised as a result).
\item Maintaining the appropriate set of server listens to ensure we are always subscribed to the correct set of paths and queries to satisfy the current set of user event callbacks (listens are started/stopped using the provided listen\+Provider).
\end{DoxyItemize}

NOTE\+: Although \doxylink{class_sync_tree}{Sync\+Tree} tracks event callbacks and calculates events to raise, the actual events are returned to the caller rather than raised synchronously.
