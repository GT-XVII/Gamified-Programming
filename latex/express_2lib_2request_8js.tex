Check if the given {\ttfamily \doxylink{_0dgoogle-cloud_2storage_2node__modules_2mime_2cli_8js_a2807c8a03bea7dd9d4118326348f457e}{type(s)}} is acceptable, returning the best match when true, otherwise {\ttfamily undefined}, in which case you should respond with 406 "{}\+Not Acceptable"{}.

The {\ttfamily type} value may be a single MIME type string such as "{}application/json"{}, an extension name such as "{}json"{}, a comma-\/delimited list such as "{}json, html, text/plain"{}, an argument list such as {\ttfamily "{}json"{}, "{}html"{}, "{}text/plain"{}}, or an array {\ttfamily \mbox{[}"{}json"{}, "{}html"{}, "{}text/plain"{}\mbox{]}}. When a list or array is given, the {\itshape best} match, if any is returned.

Examples\+: \begin{DoxyVerb}// Accept: text/html
req.accepts('html');
// => "html"

// Accept: text/*, application/json
req.accepts('html');
// => "html"
req.accepts('text/html');
// => "text/html"
req.accepts('json, text');
// => "json"
req.accepts('application/json');
// => "application/json"

// Accept: text/*, application/json
req.accepts('image/png');
req.accepts('png');
// => undefined

// Accept: text/*;q=.5, application/json
req.accepts(['html', 'json']);
req.accepts('html', 'json');
req.accepts('html, json');
// => "json"
\end{DoxyVerb}

Parse Range header field, capping to the given {\ttfamily size}.

Unspecified ranges such as "{}0-\/"{} require knowledge of your resource length. In the case of a byte range this is of course the total number of bytes. If the Range header field is not given {\ttfamily undefined} is returned, {\ttfamily -\/1} when unsatisfiable, and {\ttfamily -\/2} when syntactically invalid.

When ranges are returned, the array has a "{}type"{} property which is the type of range that is required (most commonly, "{}bytes"{}). Each array element is an object with a "{}start"{} and "{}end"{} property for the portion of the range.

The "{}combine"{} option can be set to {\ttfamily true} and overlapping \& adjacent ranges will be combined into a single range.

NOTE\+: remember that ranges are inclusive, so for example "{}\+Range\+: users=0-\/3"{} should respond with 4 users when available, not 3.
