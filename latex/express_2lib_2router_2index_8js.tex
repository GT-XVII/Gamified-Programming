Map the given param placeholder {\ttfamily name}(s) to the given callback.

Parameter mapping is used to provide pre-\/conditions to routes which use normalized placeholders. For example a {\itshape \+:user\+\_\+id} parameter could automatically load a user\textquotesingle{}s information from the database without any additional code,

The callback uses the same signature as middleware, the only difference being that the value of the placeholder is passed, in this case the {\itshape id} of the user. Once the {\ttfamily \doxylink{protobufjs_2dist_2protobuf_8js_adc8497f1b8fa9c7bee4a53c25c244a66}{next()}} function is invoked, just like middleware it will continue on to execute the route, or subsequent parameter functions.

Just like in middleware, you must either respond to the request or call next to avoid stalling the request.

\doxysubsubsection{\texorpdfstring{use}{use}}
{\footnotesize\ttfamily \label{express_2lib_2router_2index_8js_a9490b1be8fd690a8dc06d0a02805e1c9} 
\mbox{\hyperlink{chalk_2source_2index_8js_a391e164e296dc4d66cb9a11c050f2f3d}{proto}} use}

Use the given middleware function, with optional path, defaulting to "{}/"{}.

Use (like {\ttfamily .all}) will run for any http METHOD, but it will not add handlers for those methods so OPTIONS requests will not consider {\ttfamily .use} functions even if they could respond.

The other difference is that {\itshape route} path is stripped and not visible to the handler function. The main effect of this feature is that mounted handlers can operate without any code changes regardless of the "{}prefix"{} pathname.
